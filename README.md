## 🎯 목표 시나리오 예시
    - 대기열을 통과한 후 좌석 예약 및 결제를 통한 콘서트 예약
        1) 좌석 예약을 원할 경우 사용자는 특정 콘서트에 대한 대기열 토큰을 발급받는다
        2) 이 경우, 대기열 토큰의 상태 값이 ACTIVE (활성) 인 사용자만 좌석 조회 / 예약 / 결제가 가능하게 된다
        3) 좌석을 성공적으로 예약 (임시 배정) 시, TTL이 설정되며 다른 사용자는 해당 좌석을 예약할 수 없게 된다
        4) 예약 상태인 좌석을 시간 내에 성공적으로 결제를 수행되면 예약 상태는 RESERVED (예약 완료) 로 변경된다
        5) 제한 시간 내에 좌석 결제가 실패한 경우 예약 상태는 RESERABLE (예약 가능) 로 변경되며, 다른 사용자가 해당 좌석에 접근할 수 있게 된다
        6) 좌석 결제 시 포인트가 부족한 경우 충전 후 결제 과정이 수행될 수 있도록 한다
